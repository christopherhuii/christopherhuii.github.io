Blog #10: Ruby: Variable Scopes


In Algebra, variables serve as placeholders for other values. The use of variables in Algebra is no different from the use of variables in Ruby. Variables in Ruby serve as placeholders for data. However, there are four different type of variables in Ruby. These four are: local global, instance, and class variables. These four different types of variables are necessary in Ruby because they each have different scopes. Scope of a variable determines where a variable is accessible in a program. I will go into more detail of the scope of each variable in the following section.

The first variable we will talk about is the Global Variable. Global variables are created with a $ sign as its prefix. For example:

$globe = “This is a global variable”

In the example above, I created a global variable called ‘globe’ that stores the string “This is a global variable”. Global variables can be accessed from anywhere in a Ruby program. Global variables are visible throughout an entire program and can be changed anywhere throughout a program. Because global variables’ scopes are so broad, the use of global variables are strongly discouraged. Generally, programmers want everything to be as closed as possible. This helps future debugging because it is easier to narrow down the scope of the problem. Obviously, global variables still exist for a reason. Be sure to use them sparingly and wisely in order the help the sustainability of your program as it grows.

The second variable is the Local Variable. Local variables can be constructed with a lowercase letter, underscores, or digits. There is no prefix to declare a local variable. The look of a local variable is completely dependent on the person writing a code. If somebody were making a local variable that store the value of a first name, they may declare the variable name as, “first_name”. Personally, I prefer to avoid underscores and use capitalization to separate words like, “firstName”.

The scope of a local variable is limited to the block that the variable is declared in. Once the block ends, the variable cannot be accessed from anywhere outside the block of code. For example, if a local variable is declared in the method “firstName”, it cannot be accessed from the method “fullName” because it’s scope is limited in “firstName”. In addition to that, local variables cannot be accessed outside of the loop it is defined in. The scope of local variables is fairly limited.

The third variable is the Instance Variable. I briefly went over instance variables in a previous blog, but I’ll repeat myself again for clarity. Instance variables are prefixed with a single @ sign. @first_name and @age are examples of instance variables. Instance variables serve the purpose of storing information at a certain instance of an object. Instance variables’ scopes are limited to the class that they are defined in. With that said, instance variables can be accessed in multiple methods as long as they are all contained within the same class. Instance variables are able the store the information during each instance that it is being used. Objects can have their own local copy of the instance variable and their copy is independent of whatever changes are being made in the class. Instance variables are useful because it allows the scope to be wide enough for an entire class.

The fourth and last variable is the Class Variable. Class variables are prefixed with a double @@ sign. For example, @@first_name and @@age are class variables. Class variables are shared amongst all instances of a class. This is very similar to an instance variable, but class variables do not store each individual instance of an object. One variable exists for all the objects inside a class. Once one object changes the variable, it is changed for all objects. A class variable’s scope is limited to the class that it is defined in. A simple way of thinking of it is that a class variable is a global variable within the class that it is defined in.

We have gone over four different variables supported by Ruby. They might be a little confusing now, but the only way to clear up the confusion is to use them and understand their differences in your own code. I will briefly recap the scope of these variables below.

Global Variables can be accessed from anywhere in a program.
Local Variables can be accessed ONLY from where they are defined (e.g. Method or Loop).
Instance Variables can be accessed from anywhere within a class and each object can store their own instance of the variable.
Class Variables can be accessed from anywhere within a class, but there is only a single instance of the variable that is shared between all the objects.
